Before you trust this dashboard

    

      This tool joins data from two Explorer services. Both must be up, reachable, and polling your lab
      before port-watch numbers mean anything.
    


    

      We need this data for accurate reporting.
      Missing chassis or IxNetwork servers in the explorers produces incomplete joins, wrong
      isHogged results, and silent gaps in the tables below.
    


    
Required backends

    
    
      
  • 
        IxiaInventoryExplorer — chassis port inventory
        (owner, transmitState, physical port list).
      
    • 
      
  • 
        IxNetworkSessionsExplorer — IxNetwork session ports
        (cp, dp, utilization, session assignment).
      
    • 
    

    

      Configure their base URLs in this project's .env
      (INVENTORY_EXPLORER_URL, SESSION_EXPLORER_URL).
      The dashboard triggers on-demand polls on both services before each refresh (unless you start the server with --no-refresh).
    


    
What you must register in each explorer

    
    
      
  • 
        IxiaInventoryExplorer — add every chassis in your lab
        that should appear in utilization / hogging reports.
      
    • 
      
  • 
        IxNetworkSessionsExplorer — add every IxNetwork Web API server
        (IxNetworkWebAPI endpoints) that runs sessions you care about.
      
    • 
    

    

      If a chassis or server is not registered, its ports never enter the join — you may see
      "no blocked ports" when the lab is actually hogged, or N/A session fields
      for owned ports that are only visible on inventory.
